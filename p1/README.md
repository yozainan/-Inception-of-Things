# 🚀 Inception-of-Things — Part 1: K3s and Vagrant

## Overview

Part 1 sets up a **two-node K3s cluster** using Vagrant and VirtualBox:

| Machine | Hostname | IP | Role | K3s Mode |
|---------|----------|----|------|----------|
| Server | `yozainan-S` | `192.168.56.110` | Controller | `server` |
| ServerWorker | `yozainan-SW` | `192.168.56.111` | Worker | `agent` |

Both VMs run **Debian 12 (bookworm64)** with **1 CPU** and **1024 MB RAM** (configurable to 512 MB).

## ⚡ Quick Start

### Prerequisites

- **VirtualBox 7.1+** installed
- **Vagrant 2.x+** installed
- At least **4 GB free RAM** on host

### Boot the Cluster

```bash
cd p1/

# Default: 1024 MB RAM per VM
vagrant up 2>&1 | tee logs.txt

# Or with 512 MB RAM (lower resources)
VM_MEMORY=512 vagrant up 2>&1 | tee logs.txt
```

### Verify the Cluster

```bash
# Quick check — should show 2 Ready nodes
vagrant ssh yozainan-S -c "kubectl get nodes -o wide"
```

Expected output:
```
NAME           STATUS   ROLES                  AGE   VERSION        INTERNAL-IP      ...
yozainan-S     Ready    control-plane,master   Xm    v1.34.x+k3s1   192.168.56.110  ...
yozainan-SW    Ready    <none>                 Xm    v1.34.x+k3s1   192.168.56.111  ...
```

### Run Automated Tests

```bash
cd p1/
bash testing_part1.sh
```

## 📋 Step-by-Step Setup Guide

### Step 1: Install Prerequisites (host machine)

```bash
# Check if already installed
VBoxManage --version     # Should show 7.1.x
vagrant --version        # Should show 2.x.x
```

If not installed, see `plan.md` → STEP 0 for full install commands.

### Step 2: Create the VMs

```bash
cd p1/
vagrant up 2>&1 | tee logs.txt
```

This will:
1. Download the `debian/bookworm64` box (first time only, ~500 MB)
2. Create and boot **yozainan-S** → install K3s in server mode
3. Create and boot **yozainan-SW** → install K3s in agent mode, join cluster
4. Save all output to `logs.txt`

### Step 3: Verify Everything Works

```bash
# Check nodes
vagrant ssh yozainan-S -c "kubectl get nodes -o wide"

# Check hostnames
vagrant ssh yozainan-S -c "hostname"       # → yozainan-S
vagrant ssh yozainan-SW -c "hostname"      # → yozainan-SW

# Check IPs
vagrant ssh yozainan-S -c "ip a show enp0s8"    # → 192.168.56.110
vagrant ssh yozainan-SW -c "ip a show enp0s8"   # → 192.168.56.111

# Check K3s services
vagrant ssh yozainan-S -c "systemctl status k3s --no-pager"
vagrant ssh yozainan-SW -c "systemctl status k3s-agent --no-pager"

# Check resources
vagrant ssh yozainan-S -c "free -m"        # RAM
vagrant ssh yozainan-S -c "nproc"          # CPU (should be 1)
```

### Step 4: Run Tests

```bash
bash testing_part1.sh
```

All tests should show ✅ PASS.

## 🔧 How It Works

### Vagrantfile

- Defines two VMs using `debian/bookworm64`
- Each VM gets 1 CPU and configurable RAM (default 1024 MB)
- `yozainan-S` is defined first → Vagrant boots it before `yozainan-SW`
- Both get a `private_network` with static IPs in the `192.168.56.0/24` range

### server.sh (runs on yozainan-S)

1. Installs `curl`
2. Creates 1 GB swap file (prevents OOM with low RAM)
3. Installs K3s in **server mode** bound to `192.168.56.110`
4. Waits for K3s to be Ready (up to 120 seconds)
5. Exports the join token to `/vagrant/confs/node-token`
6. Sets up `kubectl` for the `vagrant` user
7. Generates an SSH key and exports the public key

### worker.sh (runs on yozainan-SW)

1. Waits for the join token from the server
2. Installs `curl`
3. Creates 1 GB swap file
4. Installs K3s in **agent mode** using the token to join `yozainan-S`
5. Accepts the server's SSH public key for cross-VM access

### Token Passing Mechanism

The K3s join token is passed from server to agent via Vagrant's shared folder:
```
server.sh saves → /vagrant/confs/node-token → worker.sh reads
```
This shared folder maps to `p1/` on the host machine.

## 🐛 Troubleshooting

| Problem | Solution |
|---------|----------|
| VirtualBox not found | Install VirtualBox 7.1 (see plan.md STEP 0) |
| `vboxdrv` module error | `sudo modprobe vboxdrv` or install kernel headers |
| Agent can't join | Check token: `vagrant ssh yozainan-S -c "cat /var/lib/rancher/k3s/server/node-token"` |
| Node stuck `NotReady` | Wait 2-3 min, then check: `vagrant ssh yozainan-S -c "sudo systemctl status k3s"` |
| OOM kills (512 MB) | Use `VM_MEMORY=1024 vagrant up` |
| Network conflict | `VBoxManage hostonlyif create` |
| Port collision | Vagrant auto-resolves (run `vagrant port yozainan-SW`) |

## 📝 Key K3s Flags Explained

| Flag | Purpose |
|------|---------|
| `--write-kubeconfig-mode 644` | Makes kubeconfig readable without sudo |
| `--node-ip` | Uses private IP instead of NAT interface |
| `--bind-address` | API server listens on private IP |
| `--advertise-address` | Nodes connect via this IP |
| `--disable traefik/servicelb/metrics-server` | Saves memory (not needed in P1) |
| `--kubelet-arg=fail-swap-on=false` | Allows K3s with swap enabled |
